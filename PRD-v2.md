# TurkerAI — Product Requirements Document (PRD) v2.0\n\n**Version:** 2.0  \n**Date:** April 19, 2026  \n**Status:** Draft — Post-Audit Revision  \n**Product:** TurkerAI Browser Extension SaaS\n\n---\n\n## 1. Overview\n\n### 1.1 Product Definition\n\n**TurkerAI** is a Chrome Extension subscription SaaS that helps microtask workers on Amazon Mechanical Turk (MTurk) and Clickworker automate the *execution* of repetitive tasks — particularly surveys with Likert scales, data entry, and transcription — using AI browser agents that operate locally within the worker\'s browser.\n\n**Core differentiation vs. existing tools:** All current tools (BZTurk, Queuebicle, TurkerHub) focus on *finding* and *accepting* tasks. TurkerAI is the first to automate actual *task execution* (filling forms, answering questions, submitting) with a human-in-the-loop, focusing on compliance and cost-effectiveness.\n\n### 1.2 Problem Statement\n\n| Worker Reality | Data |\n|----------------|------|\n| Average MTurk earnings | **$2–6/hour** (likely lower now) |\n| Workers earning above federal minimum wage ($7.25/hr) | **Only 4%** |\n| Top frustration: \"bubble hell\" (endless radio buttons) | **37.84%** of workers |\n| Repetitive survey questions | **24.16%** |\n| Tool-assisted experienced workers earn | **$8–12/hour** |\n\nThe gap between unaided and tool-assisted earnings is massive — but even existing tools don\'t close it. TurkerAI closes it by focusing on ethical, compliant automation that saves workers time and boosts earnings without relying on external cloud browser services.\n\n### 1.3 Solution\n\nAn AI agent that:\n1. **Captures** the task page (DOM, instructions, requester info) locally within the browser.\n2. **Analyzes** it via LLM — classifies task type, estimates hourly rate, assesses automation risk, and emits a recommendation (auto-execute / pre-fill / skip).\n3. **Decides** action based on worker-configured thresholds (auto-execute / pre-fill / skip).\n4. **Executes** filling fields, handling attention checks, and submitting via *local DOM manipulation*.\n5. **Logs** results for earnings analytics and approval rate tracking.\n6. **Ensures safe state recovery** on any execution failure to protect worker\'s approval rate and data.\n\n---\n\n## 2. Product Vision\n\n> *\"Every click a worker saves is money in their pocket. TurkerAI saves thousands, compliantly.\"*\n\nTurkerAI positions itself as the **first AI-native work tool** for the microtask economy — not a hit-catcher, but an AI coworker that does the tedious work so humans can focus on higher-value tasks, with a strong emphasis on compliance and ethical use. Workers always have final say.\n\n---\n\n## 3. User Profiles\n\n### 3.1 Primary User — The Automation-First Worker\n\n- **Age:** 20–45\n- **Platform:** MTurk (primary), Clickworker (secondary)\n- **Behavior:** Works 20–40 hrs/week on microtasks; uses multiple tools; seeks efficiency\n- **Pain:** Spends 40%+ of time on tedious \"bubble hell\" surveys\n- **Willingness to pay:** $10–20/month if ROI is clear (realistic: 1.5x earnings improvement)\n- **Tech literacy:** Medium — comfortable with browser extensions, can configure settings\n\n### 3.2 Secondary User — The Power Worker\n\n- **Age:** 25–55\n- **Behavior:** MTurk as primary income; uses scripts, VPNs, multiple accounts\n- **Pain:** Approval rate management, qualification maintenance, batch grinding\n- **Willingness to pay:** Up to $50/month for edge\n- **Tech literacy:** High — wants advanced config, API access, custom scripts\n\n### 3.3 Tertiary User — The Casual Worker\n\n- **Behavior:** MTurk as side income; few hours/week\n- **Pain:** Low pay, confusing UI, rejections\n- **Willingness to pay:** $5–10/month\n- **Value:** Automated pre-fill reduces cognitive load\n\n---\n\n## 4. Feature Requirements\n\n### 4.1 MVP Features (P0)\n\n#### P0.1 — Task Capture Engine\n- Inject content script on MTurk/Clickworker task pages\n- Extract: page DOM, task instructions text, form fields, requester name/ID, reward, estimated time, TurkerView reputation score\n- Send structured payload to an `offscreen document` (using `chrome.offscreen` API) for long-running processes.\n\n#### P0.2 — AI Task Analyzer (LLM)\n- Use **Gemini 3 Flash** (lowest cost: $0.72/1M input, $4.20/1M output) for task classification and basic risk assessment.\n- Use **Gemini 2.5 Pro** for nuanced attention check detection.\n- Prompt: classify task type, estimate hourly rate, assess automation risk, emit recommendation (auto-execute / pre-fill / skip).\n- Cost target: Maximize DOM parsing for free, reserve LLM for complex (0.02-0.05/task) inference only.\n\n#### P0.3 — Human-in-the-Loop Decision Engine\n- Worker configures: minimum hourly rate threshold, minimum requester reputation, automation level preference (full auto / pre-fill / manual only).\n- Three modes:\n  - **Auto-Execute:** Agent fills and submits if thresholds pass, *with explicit worker opt-in and clear ToS warnings*.\n  - **Pre-Fill:** Agent fills fields, worker reviews and submits (recommended default).\n  - **Manual:** Pass-through, no AI involvement.\n\n#### P0.4 — Local Browser AI Execution Agent\n- **No Browser Use Cloud API.** Instead, execute via *local DOM manipulation using content scripts*.\n- This allows for full control, eliminates per-user cloud browser costs, and mitigates MV3 service worker timeouts (using `chrome.offscreen` for coordination).\n- Task: navigate page elements, fill inputs, handle attention checks, click submit.\n- Screenshot capture before/after for worker review.\n- **Safe State Recovery:** On any execution failure, immediately stop, screenshot current state, alert worker, and never auto-submit a partially-filled form.\n\n#### P0.5 — Earnings Dashboard\n- Per-session earnings, hourly rate approximation\n- Approval rate tracker\n- Task type breakdown (survey vs. data entry vs. transcription)\n- Export to CSV\n\n#### P0.6 — Subscription & License Manager\n- Freemium model: 50 free AI analyses/day, 10 free AI executions/day, then subscription.\n- Tiers: Free, Pro ($14.99/mo), Power ($29.99/mo), Elite ($49.99/mo).\n- **Supabase Auth** for worker accounts and license validation via backend API.\n- API-gated AI execution (acknowledging this is a form of DRM).\n\n---\n\n### 4.2 P1 Features (Post-MVP)\n\n| Feature | Description | Priority Rationale |\n|---------|-------------|-------------------|\n| **TurkerView Integration** | Auto-fetch requester reputation from TurkerView API | Reduces risk of bad requesters |\n| **Attention Check Detector** | LLM flags suspicious attention checks before execution | Protects approval rate |\n| **Batch Auto-Accept** | Queue multiple HITs, AI analyzes and executes sequentially | Power user demand |\n| **Clickworker Support** | Extend pipeline to Clickworker UHRS platform | Market expansion |\n| **Whisper Transcription** | Audio tasks auto-transcribed via Whisper API before submission | High-ROI task type |\n| **Open-Text AI Writer** | Generate plausible open-text responses for survey comments | Medium risk — see ToS |\n| **Mobile Companion App** | Simple status dashboard on phone (PWA) | Retention feature |\n\n---\n\n### 4.3 P2 Features (Future)\n\n- Custom LLM model fine-tuned on MTurk task patterns\n- Multi-account management with session isolation\n- VPN/proxy integration for IP management\n- HIT finder with AI task quality scoring\n- Community presets (worker-shared automation configs)\n\n---\n\n## 5. Technical Architecture\n\n### 6.1 Extension Architecture\n\n```\nturkerai-extension/\n├── manifest.json              # Chrome Extension manifest v3\n├── background/\n│   ├── service_worker.js      # Main background process (Plasmo)\n│   └── offscreen.html         # For long-running tasks, chrome.offscreen API\n├── content/\n│   ├── inject.js               # DOM capture + field injection, local execution\n│   ├── page_monitor.js        # Detects new task pages\n│   └── dom_parser.js          # Extracts structured task data\n├── popup/\n│   ├── popup.html\n│   ├── popup.js               # Worker dashboard UI (React)\n│   └── popup.css\n├── options/\n│   ├── options.html           # Settings page (React)\n│   ├── options.js\n│   └── options.css\n├── lib/\n│   ├── gemini_client.js       # Google Gemini API client (proxy via backend)\n│   └── storage.js             # chrome.storage wrapper\n├── assets/\n│   ├── icon16.png, icon48.png, icon128.png\n│   └── logo.svg\n└── locales/\n    └── en/messages.json\n```\n\n### 6.2 Backend Architecture\n\n```\nturkerai-backend/\n├── app.py                     # FastAPI\n├── routes/\n│   ├── auth.py                # Supabase Auth integration\n│   ├── license.py             # POST /validate_license, API-gated execution\n│   ├── analytics.py           # POST /track_event (earnings, usage)\n│   ├── llm_proxy.py           # Proxy for Gemini API calls (security)\n│   └── subscription.py       # Stripe webhook handler\n├── models/\n│   ├── license.py\n│   └── subscription.py\n└── .env                       # SUPABASE_URL, SUPABASE_KEY, STRIPE_SECRET_KEY, GEMINI_API_KEY (backend only)\n```\n\n**Note:** Backend is a full-fledged service handling security, licensing, billing, and LLM proxying. Local execution in the browser means no Browser Use Cloud dependency.\n\n### 6.3 Tech Stack\n\n| Layer | Technology | Rationale |\n|-------|------------|-----------|\n| Extension shell | Plasmo Framework (React, MV3) | Modern MV3 toolchain, HMR, React support, offscreen document handling |\n| Background process | Service Worker (Plasmo) + chrome.offscreen | Handles MV3 timeout, long-running tasks |\n| AI Analysis | Gemini 3 Flash (classification), Gemini 2.5 Pro (attention checks) | Cost-optimized for different tasks |\n| AI Execution | Local DOM manipulation (content scripts) | Cost-free, full control, ToS compliance focus |\n| Storage | chrome.storage.sync + chrome.storage.local | Extension-native, no backend needed for core data |\n| Backend | FastAPI + PostgreSQL + Supabase Auth | Robust, scalable, secure auth, type-safe, async |\n| Payments | Stripe + Paddle | Standard for SaaS, international tax compliance |\n\n### 6.4 Key API Integrations\n\n#### Gemini API (Task Analysis) — *Proxied via Backend*\n- Endpoint: `https://generativelanguage.googleapis.com/v1beta/models/...:generateContent`\n- Auth: Backend API key, accessed by extension via session token.\n- Cost: Optimized for specific LLM tasks (classification only).\n\n#### TurkerView API (Requester Reputation)\n- Endpoint: `https://turkerview.com/api/v3`\n- Auth: Worker\'s TurkerView API key stored locally.\n- Use: Fetch requester rejection rate, pay review, speed.\n\n#### Stripe & Paddle (Payments)\n- Integrated with FastAPI backend for subscription management and webhooks.\n\n#### Supabase Auth (User Authentication)\n- Integrated with FastAPI backend for secure worker accounts.\n\n---\n\n## 7. Legal & Platform Risk\n\n### 10.1 MTurk Terms of Service\n\n**⚠️ CRITICAL RISK — MTurk ToS explicitly prohibits automated task completion.**\n\nRelevant ToS sections:\n- *\"Workers may not use bots, scripts, or other automated tools to complete HITs.\"*\n- Violation consequences: account suspension, earnings forfeiture.\n\n**Risk Mitigation:**\n1. **Human-in-the-loop as default** — Auto-execute is opt-in, not default.\n2. **Compliance-focused design** — Explicitly *not* using behavioral cloaking. If it needs cloaking, it\'s not legal.\n3. **Worker discretion** — Clear warnings in UI: \"Using auto-execute may violate MTurk ToS and CFAA.\"\n4. **Legal exposure framing** — Tool is positioned as \"AI assistant with human oversight.\"\n5. **Multi-platform** — Clickworker UHRS has more permissive ToS; prioritize there.\n6. **ToS monitoring** — Weekly automated checks on MTurk ToS pages.\n\n### 10.2 Chrome Web Store Compliance\n\n- **Deceptive practices** — Must not misrepresent what the extension does.\n- **Data collection** — Privacy policy required; minimize data collected.\n- **Payment** — Chrome Web Store payment system preferred; TurkerAI uses Stripe/Paddle (allowed if disclosed).\n\n### 10.3 Data Privacy\n\n| Data | Stored Where | Retention |\n|------|-------------|-----------|\n| License info | Backend DB (PostgreSQL) | Duration of subscription |\n| AI API keys | Backend only (proxied calls) | Never exposed to client |\n| Task analytics (earnings) | chrome.storage.local | Worker-controlled |\
| Extension usage logs | Anonymous, backend aggregate | 90 days, no PII |\n| Supabase Auth tokens | chrome.storage.local (secure) | Session-based |\n\n---\n\n## 11.
